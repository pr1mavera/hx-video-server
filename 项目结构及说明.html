# 目录结构

  • config/ ---- 项目编译配置（webpack相关，已弃用）
  • docs/ ---- 文档
  • server/ ---- 远程服务相关模块（包括视频排队、IM消息推送等接口，用于前期开发时的调试，现已弃用）
  • src/ ---- 源码
    • config/ ---- API主机配置
    • dependence/ ---- 外部依赖模块（维护小程序运行时传入的webIM协议脚本，消息推送接口等等）
    • lib/ ---- 前期开发时的调试使用到的库文件
    • module/ ---- 业务模块
      • cs$/ ---- 坐席数据流模块
      • im/ ---- IM模块
        • send/ ---- 消息发送相关
          • C2C/ ---- c2c消息
          • system/ ---- 系统消息
        • API.js ---- IM模块相关接口封装、错误处理
        • index.js ---- IM模块入口
        • login.js ---- 登录相关
        • parser.js ---- IM消息解析
      • queue/ ---- 排队模块
        • stream/ ---- 排队数据流
          • cancelQueue.js ---- 取消排队
          • heartbeat.js ---- 心跳
          • queue.js ---- 排队
          • send.js ---- 排队过程中的消息推送
          • cancelQueue.js ---- 远程socket消息接收
          • timeout.js ---- 超时（由于视频排队流程优化，此处已弃用）
        • index.ts ---- 排队模块入口
    • utils/ ---- 工具类库
    • main.js ---- 入口文件
  • rollup.config.js ---- 项目编译配置

# 模块结构

将SDK初始化好了之后，将得到一个API集合，分为三个模块

{
    im, // IM相关API
    queue, // 排队相关API
    cs // 坐席相关API
}
